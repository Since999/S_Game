.apk - 완성된 패키지
.ap_ - 7-zip 의해 만들어진 패키징
.aab - 패키지를 완성시키는 요소를 담은 패키지

.dex - 런타임에 실행되는 기계어

.class - 컴파일러에 의해 생성된 파일

bin - 이진파일 
gen - 원래 게임 카트리지 복사하고 pc에서 돌아가게 해줌
out - 컴파일 된 실행파일

.gradle - 설정 파일
bulid - 컴파일 한 코드를 실행시키는 파일

signing - 서명파일

local.properties - local sdk 위치

proguard - 사전검증 프로그램

.log - 일어난 이벤트에 대한 정보가 적혀있는 파일

captures - 스크린샷
.ipr - iProcrastinate 데이터 파일 카테고리 
.swp - 스왑파일/대비용 백업파일

.jks - 보안 인증서 파일
.keastore - 보안 인증서 파일

obj/ - 3차원 이미지가 저장된 파일 형태

.imi - 마젤란이 개발한 그래픽 파일
.iws - IntelliJ IDEA 웹 페이지

idea/ - 프로젝트의 프로젝트 별 설정 저장

.classpath - 컴파일 및 압축 파일 또는 프로젝트 종속성을 위한 프로젝트의 소스 및 대상 참조를 유지 관리
.project - 프로젝트 정보 파일
.cproject - 프로젝트 정보 파일
.settings - 프로젝트 세팅 정보 저장 파일


.war - 여러 정보를 모아 배포하는 jar파일
.ear -  서버에 배치된다

idea_modules/ - 모듈들

atlassian-ide-plugin.xml - 플러그인
idea/mongoSettings.xml - 플러그인
com_crashlytics_export_strings.xml - 플러그인
crashlytics.properties - 플러그인
crashlytics-build.properties - 플러그인
fabric.properties - 플러그인


기본 플러그인들이 들어가 있고, 사용자마다 다른 값을 가지고 있는 설정이나 경로 저장 파일들 이다.
또 대부분 빌드를 하면 사용자의 환경으로 생성되는 파일들이기 때문에 ignore를 해주는 것 이다.
